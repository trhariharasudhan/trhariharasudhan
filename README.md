## 🚀 Featured Projects

<details>
<summary><b>⚡ GrandpaOdin — Local-First Windows AI Personal Assistant</b></summary>
<br/>

A production-grade **Jarvis-style AI personal assistant** built for Windows, operating entirely on local hardware with zero cloud dependency. Designed around privacy-first principles, GrandpaOdin uses Ollama-backed open-weight models and a modular five-primitive architecture for extensible, reliable AI automation.

| Attribute | Details |
|:---|:---|
| **Stack** | Python · FastAPI · Ollama · PyO3/Rust · Tauri · TypeScript |
| **Architecture** | Five-primitive: Intelligence · Engine · Agents · Tools · Learning |
| **Performance** | Local quantized model inference, sub-second CLI response target |
| **Security** | No external API calls · Air-gapped operation · Permission confirmation system |
| **Automation** | Windows app control · Browser control · Reminder scheduler · System commands |
| **Scale** | Plugin-style skill registry · Unlimited tool extensibility · Hot-loadable agents |
| **Impact** | Full offline AI assistant — no subscriptions, no data leakage, no cloud lock-in |

GrandpaOdin implements five architectural primitives: **Intelligence** (local LLM via Ollama), **Engine** (task orchestration and routing), **Agents** (modular task executors), **Tools** (skill and plugin registry), and **Learning** (optimization and feedback pipeline). The system supports CLI and voice command modes, long-term memory, screen awareness via OCR, safe destructive-action confirmation flows, and a Tauri-based desktop frontend. Built with a Codex-compatible, sequenced engineering audit methodology for production reliability.
